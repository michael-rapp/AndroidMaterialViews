1 CircularProgressBar

The view CircularProgressBar visualizes an indeterminate progress bar as an animated circle. For further information about visualizing progress, refer to this chapter of the Material Design guidelines. As the view does not allow any user input, but is only meant to visualize a progress, its API very simple. In the following the possibilities to adapt the view's appearance programmatically or via XML attributes are discussed. The animated circle, which is shown by the view is automatically adapted to the size of the view. It is recommended to set an equal width and height, because the circle will become stretched otherwise. The resources R.dimen.circular_progress_bar_size_normal and R.dimen.circular_progress_bar_size_small provide predefined sizes for normal and small sized progress bars.

2 FloatingActionButton

The view FloatingActionButton implements a colored button, which has a shadow and an icon. For further information about this view type, refer to this chapter of the Material Design guidelines. The view provides three predefined sizes and can be shown or hidden in an animated manner. All possible sizes of the button are contained by the enum Size. The method setOnClickListener(OnClickListener):void can be used to set a listener, which should be invoked, when the button is clicked. On devices with an API level 21 or greater, a ripple effect is shown when the button is pressed. In the following sections the possibilities, which are provided to adapt the appearance of the button, will be discussed.

Showing or hiding a FloatingActionButton in an animated manner

Besides changing the visibility of a FloatingActionButton via the method setVisibility(int):void, showing or hiding the view is also possible in an animated manner. If visibility changes should be animated, the method setVisibility(int,boolean):void has to be used instead. The method's boolean parameter allows to specify, whether an animation should be used. When using the Integer parameter View.INVISIBLE, the view will become invisible, but its space is not released to be used by other views. When using the parameter View.GONE, the view's space will be released instead, after the animation has finished. The parameter View.VISIBLE allows to show a previously hidden FloatingActionButton again.
